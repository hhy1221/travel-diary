# 旅途笔记 · 本地启动与使用指南

本指南将帮助你从零开始在本地启动「旅途笔记」网站，并学会使用全部功能。

---

## 一、前提条件

请确保你的电脑已经安装以下软件：

- **Node.js**（版本 16 或以上）
- **WSL 2**（并已安装 Docker 与 MongoDB 容器）
- 已获得项目文件夹 `第六次实验`（通常位于 `D:\test\firstweb\` 或其他路径）

---

## 二、首次启动（只需完整操作一次）

### 1. 打开终端（cmd 或 PowerShell）
  按 `Win + R`，输入 `cmd` 回车；或直接打开 PowerShell。

### 2. 进入 WSL 并启动 Docker 与 MongoDB 容器
```bash
wsl
sudo service docker start
sudo docker start mongodb
```
> 如果提示容器 `mongodb` 不存在，请先跳到 **附录：创建 MongoDB 容器**，再回到这里。

检查容器是否运行成功：
```bash
sudo docker ps | grep mongodb
```
若看到 `STATUS` 为 `Up`，表示数据库已就绪。

### 3. 启动网站服务器
在 WSL 终端中进入项目目录（路径示例，请替换成你的实际路径）：
```bash
cd /mnt/d/test/firstweb/第六次实验
node server.js
```
当终端出现以下两行信息时，表示启动成功：
```
MongoDB 连接成功
服务器运行在 http://localhost:12399
```

### 4. 打开网站
在 Windows 浏览器中输入地址：  
**http://localhost:12399**

---

## 三、日常启动（后续使用只需三步）

1. 打开终端，进入 WSL：
   ```bash
   wsl
   sudo docker start mongodb
   ```
   (如果 Docker 服务未启动，先执行 `sudo service docker start`)

2. 在 WSL 中进入项目目录并运行服务器：
   ```bash
   cd /mnt/d/test/firstweb/第六次实验
   node server.js
   ```

3. 浏览器访问 `http://localhost:12399`。

## 四、功能使用详解

### 1. 注册新账号
- 点击页面右上角 **“注册”**。
- 填写用户名、邮箱、密码（两次输入需一致）。
- 点击 **“注册”** 按钮，成功后自动登录并跳转首页。

### 2. 登录
- 点击右上角 **“登录”**。
- 输入用户名和密码，点击 **“登录”**。
- 登录后，右上角显示你的用户名，并出现“写游记”、“我的”、“退出”等链接。

### 3. 首页浏览
- **横幅区**：未登录时显示注册/登录入口，登录后显示快捷功能。
- **最新游记**：按发布时间倒序显示，每页 4 篇。点击底部页码可翻页。
- **热门游记**：按浏览量从高到低展示前 4 篇。
- 点击任意游记卡片，进入详情页。

### 4. 发布游记
- 登录后，点击导航栏 **“✍️ 写游记”**。
- **游记标题**：给你的游记起一个名称（必填）。
- **目的地**：填写旅行地点，如“云南大理”（必填）。
- **旅行类型**：从下拉菜单选择（自然风光/城市漫步/美食之旅/历史文化/其他）。
- **封面图片**：可选，上传一张图片作为游记封面。
- **更多图片**：可选，按住 `Ctrl` 键多选图片，这些图片将展示在游记详情页的画廊中。
- **游记正文**：写下你的旅行故事（必填）。
- 点击 **“发布游记”**，完成后自动返回首页。

### 5. 查看游记详情
在首页或搜索结果中点击游记卡片，进入详情页，你可以看到：
- 封面大图、标题、目的地、类型、作者、发布日期
- **浏览量**：每次访问该页面，浏览量自动 +1
- **图片画廊**：如果作者上传了多张图片，会以缩略图显示，点击可查看原图
- **点赞**：登录后点击 ❤️ 按钮，红色表示已点赞，灰色表示未点赞；按钮旁显示总点赞数
- **收藏**：登录后点击 ⭐ 按钮，金色表示已收藏，灰色表示未收藏；收藏的游记会保存在个人中心
- **评论**：登录后可在页面底部输入评论并发表，所有评论按时间倒序排列
- **编辑 / 删除**：仅游记作者本人可见，点击进入相应操作

### 6. 编辑游记
- 在详情页点击 **“编辑”**。
- 可修改标题、目的地、类型、正文。
- **更换封面**：上传新图片覆盖原封面。
- **添加更多图片**：多选图片上传，会替换原有的画廊图片。
- 点击 **“保存修改”** 返回详情页。

### 7. 删除游记
- 在详情页点击 **“删除”**，确认后该游记将被永久删除。

### 8. 搜索游记
- 在页面顶部搜索框中输入关键词（可搜索标题或目的地）。
- 点击 **“搜索”**，结果将列出所有匹配的游记，并支持翻页。

### 9. 个人中心
点击导航栏 **“我的”**，进入个人中心：
- **我的游记**：展示你发布的所有游记，可快速编辑或删除。
- **我的收藏**：展示你收藏的所有游记，点击卡片可查看详情。

### 10. 退出登录
点击导航栏 **“退出”**，安全退出当前账号。

## 五、附录：创建 MongoDB 容器（仅第一次，容器不存在时）

如果在首次启动时提示容器不存在，请按以下步骤创建（在 WSL 中执行）：

```bash
sudo docker run --name mongodb -d \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=huanghanyang \
  -e MONGO_INITDB_ROOT_PASSWORD=S20061221hhy \
  mongo:7.0 --auth
```
容器创建成功后，等待几秒，再回到 **步骤一** 中的 `sudo docker start mongodb`（可忽略，因为刚创建时已自动启动），然后继续启动服务器。

## 六、常见问题

1. **MongoDB 连接失败**  
   检查 WSL 中容器是否运行：`sudo docker ps`，若未运行则执行 `sudo docker start mongodb`。如果容器丢失，按附录重新创建。

2. **端口被占用**  
   修改项目根目录下 `server.js` 文件中的 `const PORT = 12399;` 为其他端口（如 `3000`），保存后重启服务器，访问对应端口。

3. **页面样式丢失/图片加载失败**  
   确认项目中的 `public/css/style.css` 和 `public/uploads` 文件夹存在，且服务器已正确运行。若 `public/uploads` 不存在，手动创建一个空文件夹即可。

4. **无法进入 WSL**  
   确保已按课程指南安装 WSL 2 和 Debian，并已将 MongoDB 安装在 WSL 内。
